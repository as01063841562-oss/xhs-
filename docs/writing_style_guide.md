# 公众号文风指南

## 核心特点

**实战经验 + 有态度 + 不装逼**

你不是技术大牛，但你知道什么东西好用、什么是坑。你是用工具的人，不是研究理论的人。

## 语言风格

### ✅ 要这样说

- "我试了一下，发现..."（亲身实践）
- "说实话，这玩意儿..."（真实感受）
- "别被那些术语唬住了"（降低门槛）
- "我踩过的坑：..."（经验分享）
- "直接上手，别想太多"（行动导向）

### ❌ 不要这样说

- "根据最新研究表明..."（学术腔）
- "众所周知..."（装逼）
- "本文将详细阐述..."（公文体）
- "笔者认为..."（距离感）
- "综上所述..."（八股文）

## 内容结构

### 开头（痛点 + 共鸣）

用 2-3 个具体场景代入，让读者觉得"这说的就是我"。开头要有画面感，直接切入主题。

### 正文（干货 + 实操）

- 为什么要用这个？（动机）
- 怎么用？（步骤）
- 有什么坑？（避坑）
- 效果如何？（验证）
- 每 1000 字至少 1 个案例
- 理论不超过 200 字就要插案例

### 结尾（行动 + 钩子）

总结核心要点，给行动指引，留一个关注/互动的钩子。最后一句要有力量。

## 干货密度

**每篇文章至少包含：**
- 3 个可直接复用的技巧
- 1-2 个真实案例
- 1 个完整的操作流程
- 1 个避坑指南

**不要：**
- 纯概念科普（除非是入门文）
- 大段理论（超过 200 字就要插案例）
- 空洞的鸡汤

## 互动感

**多用：** 反问句、对话感、代入感
**少用：** 说教口吻、高高在上的专家范儿

## 自定义你的文风

以上只是通用建议。你可以在 `styles/writing/` 下创建自己的 yaml 文件来定义专属文风，包括人设、语气、金句、结构等。参考 `styles/writing/custom_example.yaml`。

## 示例对比

### ❌ 传统写法
```
ChatGPT 是 OpenAI 开发的大型语言模型，
基于 Transformer 架构，通过海量数据训练而成。
它能够理解和生成自然语言，在多个领域展现出卓越的能力。
```

### ✅ 推荐写法
```
ChatGPT 这玩意儿，说白了就是个超级聊天机器人。
你问它问题，它能给你靠谱的答案。
别管它背后什么原理，先用起来再说。
```

## 文章长度

- **短文（1000-1500字）：** 单一技巧，快速上手
- **中文（3000-5000字）：** 完整流程，深度实操
- **长文（6000-10000字）：** 系统方法论，多案例

**原则：** 宁可写短写透，不要写长写虚。
