# 公众号运营手册

## 文风规范

### 通用原则

1. **直接、不废话** — 说事就说事，不铺垫不绕弯
2. **口语化** — 像跟朋友聊天，不端着
3. **务实导向** — 关注能不能用、怎么用，不关心理论空谈
4. **节奏快** — 短句为主，信息密度高
5. **有主见** — 有自己的判断，不人云亦云

### 文风建议

- 用"你"不用"您"，用"咱们"不用"我们"
- 开头直接切入主题，不要"大家好我是XXX"
- 多用短句，一句话说一件事
- 技术内容要接地气，用类比和大白话解释
- 可以吐槽，可以有态度，但不抬杠
- 结尾不要"感谢阅读"，要么给行动指引，要么留个钩子
- 标题要有信息量，不做标题党但要有吸引力

### 语气示例

❌ "在当今人工智能快速发展的时代，我们有必要了解一下..."
✅ "AI 工具多到眼花，今天挑几个真正能用的聊聊。"

❌ "接下来我将为大家详细介绍这款工具的使用方法"
✅ "废话不多说，直接上手。"

## 排版风格规范

排版风格已模板化，位于 `styles/layout/` 目录，通过 `md_to_html.py --style` 参数选择。

详细的排版配色和 CSS 规范见各风格 yaml 文件。

### 通用排版规则

基础样式（每个 p 标签必带）：
```
line-height: 1.75;
letter-spacing: 0.05em;
font-size: 16px;
text-align: justify;
font-family: "PingFang SC", -apple-system-font, BlinkMacSystemFont, "Helvetica Neue", "Hiragino Sans GB", "Microsoft YaHei UI", "Microsoft YaHei", Arial, sans-serif;
```

- 图片：100% 宽度，圆角 8px，上下 margin 16px
- 每 3-4 段配一张图或一个提示框，打破阅读疲劳
- 段落之间靠 margin-bottom 自然留白

### 头图规范
- 尺寸：900 x 383（公众号推荐 2.35:1）
- 核心信息居中（383×383 安全区）
- 底部不放文字，文案≤10字

## 选题框架

### 内容分类（5 大栏目）

1. **🔧 工具测评** — 工具深度体验，说好说坏，给结论
   - 格式：工具介绍 → 实际使用 → 优缺点 → 适合谁

2. **📖 实战教程** — 手把手教你做具体的事
   - 格式：目标 → 步骤 → 踩坑 → 成果

3. **💡 效率提升** — 技巧、工作流优化、自动化方案
   - 格式：痛点 → 方案 → 效果对比

4. **🔥 热点解读** — 行业动态、新产品发布、重大事件点评
   - 格式：事件 → 解读 → 对普通人的影响

5. **🤯 脑洞实验** — 做一些有趣/奇葩的事
   - 格式：想法 → 过程 → 结果（可能翻车）

### 多条推送结构
- **头条**：当天最重要/最有价值的内容
- **次条**：轻量内容（技巧、快讯、小工具推荐）
- **三条**（可选）：实验或往期精选

## 配图规范

### 文内配图
- 工具测评：必须有工具截图（实际使用界面）
- 实战教程：关键步骤配截图或示意图
- 通用：每 600 字左右配一张图

### 注意事项
- 所有图片必须先上传到微信素材库（mmbiz.qpic.cn）
- 图片宽度统一 100%，圆角 8px
- 不用无意义的装饰图
